# ComfyPack - 작업 이어서 진행하기 위한 자료

> 이 파일은 대화 컨텍스트가 클리어된 후 작업을 이어서 진행할 때 참고하는 문서입니다.
> 마지막 업데이트: 2026-04-28 (세션2)

## 1. 프로젝트 개요

ComfyUI + comfyui_hooking_server를 Docker로 패키징하여 원클릭 배포 시스템을 만드는 프로젝트.
대상: 컴퓨터 지식 없는 일반 사용자. NVIDIA GPU 3050+ / 32GB RAM 요구.

---

## 2. 확정된 아키텍처 결정사항

| 항목 | 결정 내용 |
|------|----------|
| Docker 구조 | 완전 내장, 컨테이너 2개 분리 (ComfyUI ~15GB + hooking ~3GB) |
| 이미지 배포 | Docker Hub 공개 저장소, 계정명 `soya` (username 변경 완료) |
| 모델 파일 | 호스트 윈도우 폴더 마운트 (첫 로딩만 약간 느림, 이후 VRAM에 캐시) |
| 패치 데이터 | 커뮤니티 별도 배포 → 볼륨 마운트 |
| 워크플로우 | 커뮤니티 별도 배포 → 볼륨 마운트 |
| LLM 설정 | 설치 후 사용자가 직접 설정 |

| ComfyUI 실행 인자 | `--listen 0.0.0.0 --port 8188` 필수 |
| 모든 GitHub 저장소 공개 | Dockerfile에서 git clone 가능 |

- **Docker Hub 계정**: `soya/comfypack-*`으로 확정 (username 변경 완료)

## 4. 아직 안 한 작업 (다음에 할 것)

### 4-1. GitHub push (server.py 수정분)
- **상태**: server.py, outfit_mode.py 환경변수 지원 코드는 로컬에만 있음
- comfyui_hooking_server 저장소에 push 필요: https://github.com/lbh848/comfyui_hooking_server
- Dockerfile에서 `git clone`으로 받아오니 push 안 하면 이미지에 반영 안 됨
- `comfyui_hooking_server`의 server.py, outfit_mode.py 변경사항 push
- 저장소: https://github.com/lbh848/comfyui_hooking_server

### 4-2. Docker 이미지 빌드 + Docker Hub push
- **현재 상태**: Docker Hub에 이미지 없음 (`soya/comfypack-*` 빈 저장소)
- **순서**:
  1. `docker compose build` (30분~1시간)
  2. 빌드 성공 시 `docker compose push`로 Docker Hub 업로드
  3. 이후 사용자가 메뉴 2로 pull 가능
```bash
cd E:\soya_comfypack
docker compose build
```
- 빌드에 30분~1시간 소요 예상
- 문제 발생 시 Dockerfile 수정 필요 가능성 높음

### 4-3. ComfyUI Dockerfile 잠재적 이슈 (빌드 시 확인)
- PyTorch CUDA 12.4 인덱스 URL 확인: `https://download.pytorch.org/whl/cu124`
- `nvidia/cuda:12.4.1-runtime-ubuntu22.04` + deadsnakes PPA로 Python 3.12 설치
- 커스텀 노드 중 `requirements.txt` 없는 경우 `|| true`로 넘기게 해놓음
- `comfyui-impact-subpack`이 `comfyui-impact-pack`과 동일 repo인데 중복 clone됨 → 수정 필요
  - 현재 Dockerfile에 `ComfyUI-Impact-Pack`과 `comfyui-impact-subpack`을 같은 URL로 clone
  - subpack은 별도 repo인지 확인 필요
- 커스텀 노드 중 `requirements.txt`가 없는 경우 `|| true`로 넘기게 해놓음
- PyTorch CUDA 12.8 인덱스 URL이 실제로 존재하는지 확인 필요
  - `https://download.pytorch.org/whl/cu128` 또는 `cu126`일 수 있음
- `nvidia/cuda:12.8.0-runtime-ubuntu22.04` 이미지에 Python 3.12가 없을 수 있음 → deadsnakes PPA 추가 필요 가능

### 4-4. 실제 동작 테스트
1. 모델 폴더 마운트 후 ComfyUI에서 모델 인식 확인
2. 패치 데이터 마운트 후 hooking server 정상 동작 확인
3. 4개 워크플로우 정상 실행 확인
4. 포트 8188/8189 접속 확인

### 4-5. Docker Hub 이미지 push
- 4-2에서 빌드 성공 후 바로 진행

### 4-6. BAT 파일 실제 Windows 환경 테스트
- 스크립트 튕김 이슈는 해결됨 (3-4 참고)
- 실제 사용자 환경에서 인코딩, 경로 처리 추가 검증 필요

## 8. Docker 네트워크 구조

```
┌─────────────────────────────────────────────┐
│              Docker Compose Network          │
│                                             │
│  ┌─────────────┐    ┌──────────────────┐   │
│  │  comfyui     │    │ hooking_server   │   │
│  │  :8188       │◄───│ :8189            │   │
│  │  (~15GB)     │    │ (~3GB)           │   │
│  └──────┬──────┘    └────────┬─────────┘   │
└─────────┼────────────────────┼──────────────┘
          │                    │
     /models 마운트       8188/8189 포트 노출
     (호스트 모델 폴더)   LLM → host.docker.internal:8190
                          패치데이터 마운트
```

hooking_server는 환경변수 `REAL_COMFY_HOST=comfyui`로 Docker 서비스명으로 ComfyUI에 접근.
