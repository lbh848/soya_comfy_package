> 이 파일은 대화 컨텍스트가 클리어된 후 작업을 이어서 진행할 때 참고하는 문서입니다.

## 1. 프로젝트 개요

ComfyUI + comfyui_hooking_server를 Docker로 패키징하여 원클릭 배포 시스템을 만드는 프로젝트.
대상: 컴퓨터 지식 없는 일반 사용자. NVIDIA GPU 3050+ / 32GB RAM 요구.

---

## 4-B. 세션4 작업 내역 (2026-04-28)

### 4B-1. Docker 이미지 레이어 분석
- `docker history`로 레이어별 크기 분석 완료
- 최대 단일 레이어: PyTorch 7.25GB (10GB 미만, 권장 범위 내)
- 레이어 구조: CUDA base 3.4GB + PyTorch 7.25GB + custom_nodes 4.56GB + ComfyUI 1.04GB + 나머지

### 4B-2. 이미지 크기 증가 원인 분석
- 이전 15.4GB → 28.3GB 증가 원인: insightface 빌드 성공으로 ML 의존성 전부 설치됨
- insightface는 `comfyui-soya-custom-nodes`의 `soya_faceid_yolo_fallback.py`에서 사용 (IPAdapter FaceID)
- ML 의존성(scikit-learn, matplotlib, scipy 등) 제외 불가 → 필수 기능
- 최적화(불필요 의존성 제거) 검토 → 1-2GB 절감에 불과, 복잡도 증가로 보류

### 4B-3. 볼륨 마운트 추가
- `input/` 마운트: 사용자 참고 이미지 입력용
- `output/` 마운트: 생성 결과물 확인용
- `snapshots/` 마운트: ComfyUI-Manager 스냅샷 보존 (사용자 노드 자동 복원용)
- `user_nodes/` 마운트: 사용자 추가 커스텀 노드 영구 보존

### 4B-4. 사용자 커스텀 노드 자동 복원 시스템 구현
**문제**: 컨테이너 재생성(업데이트) 시 사용자가 추가 설치한 노드가 날아감

**해결方案**: `user_nodes/` 마운트 + 심볼릭 링크 + 스냅샷 복원

**구조**:
```
호스트 (Windows)              컨테이너 (Linux)
./user_nodes/     → 마운트 →  /app/user_nodes/
                              ↓ 심볼릭 링크
                              /app/comfyui/custom_nodes/{user_node} → /app/user_nodes/{user_node}
```

**새 파일**:
- `comfyui/entrypoint.sh` — 컨테이너 시작 시 restore_nodes.py 실행 후 ComfyUI 구동
- `comfyui/restore_nodes.py` — 4단계 복원 로직
- `comfyui/Dockerfile` — ENTRYPOINT 추가, `builtin_nodes.txt` 생성

**복원 4단계** (컨테이너 시작 시 자동):
1. **Migrate**: Manager로 설치한 노드를 `custom_nodes/` → `user_nodes/`로 이동 + 심볼릭 링크
2. **Symlink**: `user_nodes/`의 노드를 `custom_nodes/`에 심볼릭 링크
3. **Snapshot restore**: Manager 스냅샷에서 없는 노드를 `user_nodes/`에 clone + 심볼릭 링크
4. **Pip deps**: `user_nodes/`의 모든 노드 pip 의존성 설치

**장점**:
- 이미지 크기 변화 없음 (28.3GB)
- 사용자 노드, CSV, 설정파일 등 모든 파일 영구 보존
- 업데이트 시 자동 복원 (사용자 개입 불필요)

### 4B-5. 디스크 용량 요구사항 정리
